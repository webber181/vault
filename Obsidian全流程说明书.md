# Obsidian 全流程说明书

> 目标：把 Webber 的 Obsidian 知识库，做成「可编辑、可同步、可自动化、可长期维护」的一套工作流。

## 1. 总体原则

1. **iCloud 作为主库**：Mac、iPhone 都围绕同一套 iCloud vault。
2. **Markdown 为主**：所有内容都落到 `.md` 文件。
3. **模板驱动**：日记、文章、概念、项目尽量用模板创建。
4. **低摩擦写作**：先记下来，再整理。
5. **主库唯一**：iCloud 主库是唯一真源，不再维护额外本地副本。

## 2. 当前目录与路径

### 2.1 服务器端 vault
- 路径：服务器本地路径（示例：`/home/debian/vault`）
- 用途：自动化脚本写入、日报生成、知识整理、Git push

### 2.2 本地 Mac / iPhone 共享 vault
- Canonical 路径：`/Users/webber/Library/Mobile Documents/com~apple~CloudDocs/Notes-Work`
- 要求：Mac 和 iPhone 打开**同一个 vault**，不要再建第二个仓库

### 2.3 关系说明
- 服务器端负责抓取与生成
- 本地端负责阅读、编辑、补充知识
- GitHub 负责跨环境备份和版本回溯

## 3. 推荐目录结构

- `模板/`：模板
- `100-日记/`：日记、每日复盘
- `200-概念/`：概念卡片
- `300-文章/`：文章笔记
- `400-项目/`：项目笔记
- `500-领域/`：长期领域沉淀
- `LLM知识库/`：AI / 研究资料沉淀
- `AI新闻中心/`：资讯归档
- `Daily Reports/`：自动化日报（当前主目录）
- `每日报告/`：历史日报目录（兼容保留）

## 4. 核心插件

### 4.1 Templater
用途：自动填充日期、标题、分类、前置字段。

### 4.2 QuickAdd
用途：一键创建不同类型笔记。

### 4.3 Dataview
用途：查询、聚合、统计笔记。

### 4.4 Obsidian Git
用途：自动 pull + 自动备份提交，push 仍手动触发。

## 5. 日常使用流程

### 5.1 每日开始
1. 打开 Obsidian
2. 进入今日日记
3. 查看昨日未完成项
4. 把今天要做的事情写进日记

### 5.2 阅读文章时
1. 用文章模板建笔记
2. 记录来源、摘要、关键观点
3. 抽取值得复用的概念到概念卡片
4. 用 wikilink 串联相关笔记

### 5.3 做项目时
1. 建项目笔记
2. 写目标、里程碑、风险、待办
3. 项目笔记中链接相关概念、文章、日报

### 5.4 AI / 研究资料沉淀
1. 原始文章进入 `LLM知识库/raw/`
2. 再整理成概念、索引、总结
3. 保持“原文 / 摘要 / 观点 / 关联”分层

## 6. 写作规范

1. **原子化**：一篇笔记只表达一个核心观点
2. **标题清晰**：标题直接表达内容
3. **链接优先**：能链接就不要重复写一遍
4. **保留来源**：文章类内容一定保留来源链接
5. **结构固定**：同类笔记尽量使用一致模板

## 7. 模板使用建议

### 7.1 日记
建议内容：
- 今日重点
- 未完成事项
- 复盘
- 观察到的市场/项目变化

### 7.2 文章笔记
建议内容：
- 来源
- 作者
- 结论摘要
- 关键段落
- 可复用观点
- 关联笔记

### 7.3 概念笔记
建议内容：
- 定义
- 为什么重要
- 常见误区
- 相关概念
- 真实案例

### 7.4 项目笔记
建议内容：
- 目标
- 当前状态
- 里程碑
- 风险
- 下一步动作

## 8. 同步机制

### 8.1 iCloud 同步
适合 Mac / iPhone 之间的即时同步。

### 8.2 Git 同步
适合：
- 历史版本回滚
- 服务器自动写入
- 保险备份

### 8.3 推荐顺序
1. 本地编辑
2. Obsidian Git 自动备份提交
3. GitHub 远端备份
4. 服务器端定时任务拉取/写入
5. iCloud 再同步到移动端

## 9. 服务器自动化流程

### 9.1 自动写入内容
服务器侧脚本会把：
- 新闻摘要
- 每日复盘
- 知识整理
- 研究笔记
写入 vault 对应目录。

### 9.2 人工触发
如果需要手动刷新，可运行对应脚本，再检查 vault 文件是否更新。

## 10. 常见操作

### 打开 vault
- Mac 端直接打开 `Notes-Work`
- iPhone 端选择同一个 iCloud vault

### 新建一篇文章笔记
- 用 QuickAdd 或模板创建
- 填来源、摘要、观点
- 放进 `300-文章/`

### 新建一个概念卡片
- 用概念模板创建
- 放进 `200-概念/`

### 查看自动日报
- 打开 `Daily Reports/`
- 结合当天日记一起看

## 11. 常见问题

### 11.1 iPhone 不能导入 vault
通常是因为：
- 没有放在正确的 iCloud 目录下
- 误建了第二个 vault

解决方式：
- 删除错误 vault
- 重新用同一个 iCloud 路径打开

### 11.2 同步不一致
优先检查：
1. iCloud 是否登录正常
2. Obsidian Git 是否启用
3. GitHub 是否有最新提交
4. 服务器脚本是否正常写入

### 11.3 目录里找不到新文件
检查：
- 文件是不是已经写入 GitHub 对应分支
- 文件是不是写到了 `Daily Reports/` 而不是 `每日报告/`
- 本地是否还没同步下来
- Obsidian 是否缓存了旧状态

## 13. 约定

- **Ob / Obsidian**：都指 Webber 的知识库编辑环境
- **Vault**：整套 Obsidian 文件夹
- **日记**：每天记录与复盘
- **概念卡片**：原子化知识单元
- **文章笔记**：外部内容的整理入口
- **项目笔记**：任务与执行管理

## 14. 结论

这套系统的核心不是“堆很多文件”，而是：

- 统一入口
- 统一模板
- 统一同步
- 统一链接
- 统一维护

只要保持这五个统一，Obsidian 就会越来越顺手，而不是越来越乱。

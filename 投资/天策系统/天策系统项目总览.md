# 天策系统项目总览

> [!summary] 一句话概括
> 天策系统是一个 **A股多因子选股 + 市场风控 + 持仓管理 + 回测复盘** 的一体化驾驶舱。
> 它的核心链路是：**更新数据 → 校验质量 → 计算三指标与评分 → 按市场状态筛选 → 输出 Top50/Top3 → 持仓跟踪 → 回测复盘 → 参数配置**。

## 当前扫描结论

- 系统已正常接入并返回结果
- 核心数据更新到 **2026-04-21（交易日）**
- 但 `validate_data` 当前提示：**最新交易日已到 2026-04-22，当前数据日偏旧**
- 说明：**需要先做一次全量更新**，再谈当天的正式盘前/盘中决策

## 一、系统工作原理

### 1）数据来源
天策系统当前使用的核心数据源包括：

- `stock_basic`：股票基础信息
- `daily`：日线行情
- `daily_basic`：估值、流通市值等基础面数据
- `trade_cal`：交易日历
- `index_daily`：指数行情
- `moneyflow`：资金流
- `top_inst`：机构/游资龙虎资金
- `fina_indicator`：财务指标

### 2）处理流水线
系统不是单纯“选股列表”，而是按流水线跑：

1. **全量拉取/更新**
   - 股票池、日线、资金流、指数、基本面、交易日历分项更新
2. **Schema 校验**
   - 检查字段完整性、修复行、丢弃行
3. **指标计算**
   - 计算 `ST / QQE / SSL`
   - 计算 `三重共振`
   - 计算 `综合得分`
   - 计算止损/止盈
   - 计算近 3 日涨幅、近 5 日平均成交额等
4. **市场状态判断**
   - 用涨跌家数、资金净流、指数投票判断 `RISK_ON / RISK_OFF / NEUTRAL`
5. **筛选与打分**
   - 先做基础过滤，再做评分门槛，再做资金流过滤
   - 最终产出 `Top50`、`Top3`、持仓动作建议
6. **展示与交互**
   - 前端分页、筛选、导出、回测、复盘、参数配置

### 3）核心风控思想
- **市场不对，个股再强也要降级处理**
- 系统不是只看分数，而是看：
  - 市场状态
  - 三指标是否共振
  - 资金是否配合
  - 基本面是否踩雷
  - 风险收益比是否合格

### 4）当前风控参数的直观含义
从参数页可以看出，系统默认倾向于：

- **排除 ST / *ST**
- **排除科创板（688）**
- **排除解禁风险**
- **最小成交额过滤：1 亿元**
- **Top N 输出：50 只**
- **最终操作输出：Top3**
- **市场环境不佳时，仓位与评分会明显折扣**

## 二、关键 API / 运行接口

### 常用接口
- `/strategy/status`
- `/strategy/update_status`
- `/strategy/market_status`
- `/strategy/date`
- `/strategy/schema_report`
- `/strategy/moneyflow_snapshot`
- `/strategy/market_snapshot`
- `/strategy/params`
- `/strategy/validate_data`
- `/strategy/log?lines=...`
- `/strategy/results?filter_mode=strong|full&format=list|map...`
- `/strategy/refresh`
- `/strategy/update_refresh?component=...`
- `/strategy/refresh_one?code=...`

### 重要说明
- `format=list` 在结果接口里很关键，不加可能拿不到预期表格数据
- `page_size` 不能太小，系统有最小限制
- 当前扫描时，`/strategy/decision_board`、`/strategy/daily_action_report` 这两个直连接口返回 404，说明：
  - 可能已改名
  - 或前端页面改为内嵌组件，不再单独暴露该 API

## 四、标签页使用说明

> 下面按页面标签页解释，每个页签负责什么、适合干什么、怎么看。

### 1）天策精选
**定位：主驾驶舱 / 今日精选总览**

用途：
- 看当天最重要的精选股票
- 看策略精选明细和综合评分
- 看当前市场状态与执行建议
- 一键刷新策略、查看校验异常

页面里通常包含：
- 今日精选股票
- 策略精选明细
- Schema 校验异常明细
- 数据接口分项更新
- 战术指令区
- 精细漏斗区

适合做的事：
- 每天第一眼先看这里
- 快速确认“今天系统给谁”
- 看是否可以进入持仓/观察名单

---

### 2）股票明细
**定位：全市场明细表 / 精筛入口**

用途：
- 查看全市场股票的完整结果表
- 支持筛选、排序、分页、导出 CSV
- 可看核心字段：
  - 代码
  - 名称
  - 行业
  - 综合得分
  - 操作建议
  - 仓位管理
  - 现价
  - 止损/止盈
  - ST / QQE / SSL
  - 龙虎资金
  - 淘汰原因

适合做的事：
- 从全市场里找候选
- 复核某只票为什么进/出名单
- 导出结果做二次分析

---

### 3）我的自选
**定位：自选股管理区**

用途：
- 从全市场加入自选
- 维护自己的长期观察池
- 查看自选股的最新评分和动作建议

适合做的事：
- 把你关注的票长期挂进来
- 跟踪自选股是否走强/走弱
- 作为盘前、盘中重点观察列表

---

### 4）我的持仓
**定位：真实持仓监控区**

用途：
- 录入底仓
- 查看持仓数量、成本价、现价、市值、盈亏
- 看动态止损位、硬止损位
- 监控当前仓位和风险

适合做的事：
- 管理真实持仓
- 盘前判断先处理持仓还是开新仓
- 结合系统评分做风控

这里通常会看：
- 可用资金
- 总市值
- 总资产
- 总盈亏
- 累计仓位

---

### 5）回测实验室
**定位：策略验证 / 事后检验区**

用途：
- 构建目标池
- 运行带成本回测
- 查看回测结果：
  - 年化收益
  - 最大回撤
  - 夏普比率
  - 胜率
  - 交易总笔数
- 保存精选快照、更新跟踪数据

适合做的事：
- 验证参数是否有效
- 看策略是不是“真赚钱”还是“纸面强”
- 回测单次目标池，而不是盲目全市场扫

---

### 6）精选跟踪
**定位：精选批次跟踪 / 次日与中期表现观察**

用途：
- 查看历史精选批次
- 跟踪批次里的精选明细
- 看是否触发事件、是否复盘完成
- 统计次日强势率、3日爆发率、5日趋势率
- 生成 Hermes 提示词

适合做的事：
- 看“昨天选出来的票后来怎样了”
- 做批次级复盘
- 生成给大模型继续分析的上下文

---

### 7）复盘优化
**定位：单票复盘 / 经验沉淀区**

用途：
- 查看复盘池
- 对单只股票做复盘
- 打复盘标签
- 保存复盘结果
- 生成提示词草稿

适合做的事：
- 复盘成功/失败原因
- 总结为什么买、为什么没买、为什么卖
- 积累模板化经验

---

### 8）精选过程
**定位：筛选过程 / 执行链路可视化**

用途：
- 看数据修复、丢弃、异常类型
- 看数据接口分项更新
- 看战术指令区
- 看精细漏斗区
- 看全局控制区
- 看当前筛选过程中哪些票被什么规则筛掉

适合做的事：
- 排查“为什么某票没进去”
- 检查筛选链路有没有异常
- 看系统是怎么一步步缩到 Top3 的

---

### 9）数据更新
**定位：数据维护 / 重新跑数区**

用途：
- 查看各组件更新状态
- 看最后更新时间
- 单独更新某个分项
- 全量更新策略数据
- 查看日志

组件通常包括：
- 全部更新
- 日线数据更新
- 重新执行选股（仅重算）
- 龙虎资金更新
- 大盘基本面更新
- 基本面更新
- 交易锚点与质量校准

适合做的事：
- 当日数据刷新
- 定位哪一步卡住
- 强制重算本地 SQLite，不重新拉全网数据

### 10）参数配置
**定位：策略参数总开关 / 调参区**

用途：
- 查看核心技术指标参数

## 五、系统的实际使用顺序

我建议按这个顺序看：

1. **数据更新**
   - 确认今天是不是最新交易日
2. **天策精选**
   - 看当天精选和市场状态
3. **股票明细**
   - 查看候选池与筛选原因
4. **我的持仓 / 我的自选**
   - 确认真实仓位和观察池
5. **回测实验室**
   - 验证策略是否可行
6. **精选跟踪 / 复盘优化**
   - 把结果变成经验
7. **参数配置**
   - 最后再调参数，不要一上来就调

## 六、当前系统理解要点

### 1）它不是单一选股器
它实际上是一个：
- 市场状态判断器
- 三指标选股器
- 风险控制器
- 持仓管理器
- 回测验证器
- 复盘沉淀器

### 2）市场状态优先级高于个股分数
- 即使个股分数高
- 如果市场处于 `RISK_OFF`
- 也要降级为观察/防守思路

### 3）评分高不等于可立即买
还要看：
- 三维共振
- 资金流
- 止损空间
- 基本面过滤
- 近期涨幅
- 市场是否允许开仓

### 4）精选结果是“可执行建议”，不是“绝对结论”
系统输出更像：
- 今日可观察
- 今日可持有
- 今日可回避
- 今日可回测

而不是简单的“买入清单”。
